# Установка Live Transcriber на Windows 11

Программа умеет:

- распознавать речь во время онлайн-встречи
- сохранять текст в `txt`
- экспортировать результат в `docx`
- сохранять `wav`-аудио встречи для повторного прослушивания фрагментов
- искать текст по стенограмме
- ставить метки `Important`, `Task`, `Question`
- собирать краткий `Summary`
- по желанию делить поток `Meeting` на `Speaker 1`, `Speaker 2`, ...

## 1. Установите Python

1. Откройте сайт:
   https://www.python.org/downloads/windows/
2. Скачайте `Python`.
3. Во время установки поставьте галочку `Add Python to PATH`.
4. Нажмите `Install`.

## 2. Установите ffmpeg

1. Откройте `PowerShell`.
2. Вставьте команду:

```powershell
winget install Gyan.FFmpeg
```

3. Дождитесь окончания установки.
4. Закройте PowerShell.

## 3. Скопируйте папку программы

Скопируйте папку `speech_transcriber` на Windows, например сюда:

```text
C:\speech_transcriber
```

## 4. Откройте PowerShell в папке программы

1. Откройте папку:

```text
C:\speech_transcriber
```

2. Нажмите на адресную строку сверху.
3. Введите:

```text
powershell
```

4. Нажмите `Enter`.

## 5. Создайте виртуальное окружение

Вставьте команду:

```powershell
python -m venv .venv
```

## 6. Активируйте окружение

Вставьте команду:

```powershell
.venv\Scripts\activate
```

Если Windows покажет ошибку, сначала выполните:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

Потом снова:

```powershell
.venv\Scripts\activate
```

## 7. Установите зависимости

Вставьте команду:

```powershell
pip install -r requirements.txt
```

Если вы хотите использовать разделение собеседников на `Speaker 1`, `Speaker 2` и дальше, ничего отдельно ставить не нужно: нужная библиотека уже входит в `requirements.txt`.

Для воспроизведения куска аудио по нажатию на текст тоже ничего отдельно ставить не нужно: модуль для этого уже входит в `requirements.txt`.
Если один способ воспроизведения не сработает, программа попробует запасные варианты, включая `ffplay` из `ffmpeg`.

## 8. Запустите программу

Вставьте команду:

```powershell
.\run_live_transcriber.bat
```

Так надежнее, чем запускать `python live_transcriber.py` вручную:

- создастся `.venv`, если его еще нет
- автоматически поставятся зависимости
- программа запустится тем Python, в который эти зависимости установлены

## 9. Как пользоваться

1. Откройте встречу в браузере.
2. Убедитесь, что вы слышите собеседников через компьютер.
3. В программе выберите модель `base` для первого запуска.
4. Если на вашем ПК бывают пропуски звука или предупреждения `data discontinuity in recording`, оставьте включенной галочку `Stable mode`.
5. Если хотите разделять участников встречи внутри потока `Meeting`, включите галочку `Split meeting speakers`.
6. Нажмите `Start`.
7. Смотрите текст в окне.
8. Если нужно найти слово или фразу, используйте поле `Search`.
9. Можно нажимать прямо по значкам в строке: `▶`, `✓`, `?`, `❗`.
10. Если хотите прослушать кусок записи, нажмите `▶` в нужной строке. Можно также выбрать строку и нажать `Play Selected`.
11. Если нужен краткий итог встречи, нажмите `Summary`.
12. После встречи нажмите `Stop`.
13. Если нужен Word-файл, нажмите `Export DOCX`.

Что означают подписи в тексте:

- `Meeting`: общий звук встречи из колонок компьютера
- `You`: ваш микрофон
- `Speaker 1`, `Speaker 2`, ...: участники встречи, если включено разделение спикеров

Что означают значки в строках:

- `▶`: эту строку можно воспроизвести
- `✓`: нажмите, чтобы включить или снять метку задачи
- `?`: нажмите, чтобы включить или снять метку вопроса
- `❗`: нажмите, чтобы включить или снять метку важного
- `○`: у строки пока нет метки

Какие файлы создает программа:

- `live_transcripts\*.txt`: текст стенограммы
- `live_audio\*.wav`: сохраненный звук для повторного прослушивания фрагментов
- `logs\live_transcriber.log`: журнал работы

## 10. Если будет ошибка

В программе есть кнопки:

- `Save Error Report`
- `Open Logs`

Если что-то сломается, пришлите мне:

- файл отчета
- файл `logs\live_transcriber.log`

Важно про разделение спикеров:

- это необязательная функция, ее можно просто выключить
- она работает только для потока `Meeting`
- качество зависит от звука, шума и того, перебивают ли собеседники друг друга

Если звук на слабом ПК записывается с пропусками:

- включите `Stable mode`
- выберите модель `base` или `tiny`
- закройте лишние тяжелые программы
- сначала добейтесь стабильной записи, а потом уже включайте разделение спикеров

## 11. Если захотите собрать EXE

Откройте PowerShell в папке программы и выполните:

```powershell
.\build_exe.bat
```

После этого файл будет здесь:

```text
C:\speech_transcriber\dist\LiveMeetingTranscriber\LiveMeetingTranscriber.exe
```
